Developer Handoff Brief

Everything Cursor Agent (or a human developer) needs to port this design system into production. Token map, component inventory, performance budget, font and image pipelines, file structure, naming conventions, and stack-specific notes. Read this before writing a line of production code.

Version 1.0 April 2026 Stack-agnostic CSS variables + vanilla HTML/CSS/JS • Applies to: dade.studio, contentcrib.studio, maas.dade.studio, future microsites
01 / Token Map

CSS Variable Structure

All design decisions live as CSS custom properties on :root. Dark mode overrides are scoped to [data-theme="dark"] — not @media (prefers-color-scheme) alone — so user preference can override system preference via JS with no FOUC.

Critical rule
Never hardcode a color hex in a component. Always reference a CSS variable. If a token doesn't exist for what you need, add it to the token map first, then use it. This is the only rule that has no exceptions.

02 / Component Inventory

Component Inventory

Every component defined in the design system. Column "Drift risk" flags which components are most likely to be incorrectly implemented — pay extra attention to these during porting.

Button
High drift risk
6 variants (primary, secondary, ghost, cyan, yellow, purple). 4 sizes (sm/md/lg/xl). Must maintain: 2px border on all, min-height 44px, active press transform, focus ring 2.5px crimson 3px offset, disabled 0.4 opacity.
Badge / Tag
Low drift risk
JetBrains Mono, 10px, all-caps, letter-spacing 0.08-0.1em. 9 color variants. Never decorative-only — always label something semantic. Never use emoji as badge content.
Card
Medium drift risk
Always: border-radius var(--r-xl), 1px border on --border-subtle, hover lift translateY(-2px) + shadow-lg + border-default. Accent bar is 4px height, full width, any brand color. Footer background --bg-subtle.
Callout
Low drift risk
4px left border, no full border. role="note" for informational, role="alert" for warning/error. Icon is always decorative (aria-hidden). 5 variants matching semantic states + brand.
Form inputs
High drift risk
Must: min-height 44px, explicit label with for/id, focus ring 3px crimson glow, error state aria-invalid + aria-describedby, never label-less with placeholder only. -webkit-appearance: none on mobile.
Navigation
High drift risk
Fixed positioning, backdrop-filter blur(16px). Skip link in DOM before nav. Mobile menu: aria-expanded on trigger, role="dialog" on drawer, Escape key closes, body overflow:hidden when open. aria-current="page" on active link.
Data table
Medium drift risk
Always: caption, scope="col" on headers, overflow-x:auto on wrapper (never reflow to cards). Complex tables need headers attribute on cells. Row hover on tbody only.
Logo mark
High drift risk
Always an <img> tag pointing at /assets/brand/logo-d.png. Canonical asset is a 1,425x1,425 PNG — raster is source of truth, no SVG variant exists, never regenerate from prompts. Set explicit width/height attributes and meaningful alt ("Dade Studio"). The parent link carries aria-label if the mark stands alone.
Image treatment
Medium drift risk
Always picture with AVIF + WebP sources. Explicit width/height attributes. aspect-ratio on container. fetchpriority="high" on single LCP image only. loading="lazy" below fold.
Icons
Low drift risk
Lucide icon set, inline SVG only. 24x24 viewBox, stroke-width 2.5, currentColor. Decorative: aria-hidden="true". Interactive standalone: aria-label on parent button. Never icon fonts, never emoji as UI.
FAQ accordion
Medium drift risk
CSS max-height transition (not JS height animation). aria-expanded on trigger button. One open at a time pattern. Escape key closes. No layout shift on toggle — max-height must be set high enough.
Pull quote
Low drift risk
4px left border in brand color. blockquote element. Attribution in cite or separate element, never as plain p. Large decorative quote mark via ::before — aria-hidden or kept presentational.
03 / Performance Budget

Performance Budget

These targets are non-negotiable. Measure on Lighthouse mobile simulation (Moto G4, 4G throttled) before any PR merges to main. If a page misses the budget, it does not ship.

LCP < 2.0s
Largest Contentful Paint on 4G mid-tier mobile
CLS < 0.05
Cumulative Layout Shift — no font swap shift, no image shift
INP < 200ms
Interaction to Next Paint — no long tasks blocking main thread
TBT < 100ms
Total Blocking Time — no large JS bundles on critical path
JS budget < 30KB
Gzipped per page — JS is progressive enhancement only
Lighthouse 100/100/100/100
Perf / A11y / Best Practices / SEO — all four, always
What kills the budget most often
Third-party scripts on the critical path (analytics, chat widgets, tag managers). Unoptimized hero images without explicit dimensions. Render-blocking stylesheets via @import. Missing font preloads causing late FOUT. Adding any of these without measuring first will blow the budget.
04 / Font Pipeline

Font Loading Strategy

Two variable fonts, self-hosted from /assets/fonts/. One WOFF2 file each. Zero third-party DNS lookups, zero multiple-weight requests. This is the complete loading strategy — do not deviate from it or CLS will spike.

head — font loadingHTML
<!-- Step 1: Preload the self-hosted WOFF2 files before any other request -->
<link rel="preload"
      href="/assets/fonts/InterVariable.woff2"
      as="font" type="font/woff2" crossorigin>
<link rel="preload"
      href="/assets/fonts/JetBrainsMono[wght].woff2"
      as="font" type="font/woff2" crossorigin>

<!-- Step 2: Declare @font-face blocks in the stylesheet (see type.css) -->
<!-- No external stylesheet link needed. No preconnect needed. -->
Self-hosted, not CDN
Both variable fonts ship from /assets/fonts/ on the same origin as the HTML. Source: Inter Variable from rsms.me/inter/, JetBrains Mono variable from the JetBrains fonts repo. No Google Fonts, no third-party origins, no preconnect handshakes. Same font-display: swap strategy applies.

@font-face + Size-Adjust for Zero CLS

Declare both variable fonts with format("woff2-variations"). Add size-adjust, ascent-override, and descent-override to a fallback @font-face so the fallback matches Inter's metrics. This eliminates layout shift during the swap period. These declarations live outside @layer tokens (@font-face cannot be cascade-layered).

type.cssCSS
/* Self-hosted variable fonts — place outside @layer tokens */
@font-face {
  font-family: "Inter";
  src: url("/assets/fonts/InterVariable.woff2") format("woff2-variations");
  font-weight: 100 900;
  font-display: swap;
}
@font-face {
  font-family: "JetBrains Mono";
  src: url("/assets/fonts/JetBrainsMono%5Bwght%5D.woff2") format("woff2-variations");
  font-weight: 100 800;
  font-display: swap;
}

/* Fallback that matches Inter's metrics — eliminates CLS */
@font-face {
  font-family: "InterFallback";
  src: local('Arial');
  ascent-override:  90%;
  descent-override: 22%;
  line-gap-override: 0%;
  size-adjust:      107%;
}

/* Primary family stack — Inter leads, calibrated fallback next */
--font-sans: "Inter", InterFallback, ui-sans-serif, system-ui, sans-serif;

05 / Image Pipeline

Image Pipeline

Every image in production must go through this pipeline. No exceptions for "it's just a small image" — small images still cause CLS if they're missing dimensions.

Format Requirements

FormatUse caseToolQuality setting
AVIFPrimary format for all photography, illustrations, screenshotsSquoosh, sharp, @squoosh/libQuality 60-75 for photos, 80 for UI screenshots
WebPFallback for browsers without AVIF (Safari <16)cwebp, sharp, imagemin-webpQuality 80-85
PNGFinal fallback only, or logos/icons with transparency that aren't SVGoxipng, pngquantLossless or quality 80+
SVGAll logos, icons, geometric illustrationsSVGOMinified, viewBox preserved, presentational attrs removed
JPEGNot used as a source format — convert everything to AVIF+WebP----

Markup Pattern

Do
<picture> <source srcset="hero-640.avif 640w, hero-1280.avif 1280w" type="image/avif"> <source srcset="hero-640.webp 640w, hero-1280.webp 1280w" type="image/webp"> <img src="hero.jpg" width="1280" height="720" alt="Descriptive alt text" loading="eager" fetchpriority="high" decoding="async"> </picture>
Don't
/* Missing: dimensions, AVIF, WebP */ <img src="hero.jpg" alt="Hero image" > /* Also wrong: fetchpriority on non-LCP images causes priority inversion — use sparingly */ <img src="logo.png" fetchpriority="high" >

Responsive srcset Rules

R1
Generate srcset widths at: 320, 640, 768, 1024, 1280, 1536, 1920px. Never generate sizes larger than the source image.
R2
Add the sizes attribute whenever the image isn't full viewport width. Example: sizes="(max-width: 1024px) 100vw, 50vw" for a 2-col grid image.
R3
fetchpriority="high" goes on exactly one image per page — the LCP element. On interior pages this is usually the case study hero. On the homepage it's the hero background or the largest visible image above the fold.
R4
All images below the fold get loading="lazy". Images in the initial viewport get loading="eager". Never set loading="lazy" on LCP.
R5
Every image container must have aspect-ratio set in CSS matching the image's natural ratio. This reserves space before the image loads and prevents CLS regardless of font or network conditions.

  /* One-off overrides, .sr-only, .truncate, layout helpers */
  /* Utilities win over components without !important */

Dark Mode Implementation

theme.jsJS
// Complete theme system — place in <script> before </body>
(function () {
  const SUN  = '<circle cx="12" cy="12" r="5"/><path d="M12 1v2..."/>';
  const MOON = '<path d="M21 12.79A9 9 0 1 1 11.21 3 7 7 0 0 0 21 12.79z"/>';

  function applyTheme(t) {
    document.documentElement.setAttribute('data-theme', t);
    const isDark = t === 'dark';
    const btn = document.getElementById('theme-btn');
    if (btn) {
      btn.setAttribute('aria-pressed', String(isDark));
      document.getElementById('theme-label').textContent = isDark ? 'Dark' : 'Light';
      document.getElementById('theme-icon').innerHTML = isDark ? MOON : SUN;
    }
  }

  // Load order: localStorage → system preference → default light
  const saved = localStorage.getItem('ds-theme');
  const sys   = window.matchMedia('(prefers-color-scheme: dark)').matches;
  applyTheme(saved || (sys ? 'dark' : 'light'));

  // Respond to OS-level changes only when no manual override is stored
  window.matchMedia('(prefers-color-scheme: dark)')
    .addEventListener('change', e => {
      if (!localStorage.getItem('ds-theme')) applyTheme(e.matches ? 'dark' : 'light');
    });

  window.toggleTheme = () => {
    const next = document.documentElement.getAttribute('data-theme') === 'dark'
      ? 'light' : 'dark';
    localStorage.setItem('ds-theme', next);
    applyTheme(next);
  };
})();
FOUC prevention
Set data-theme on <html> before the document paints. In Next.js, use the script prop on _document with strategy="beforeInteractive". In static HTML, place the theme-reading script in a <script> tag immediately after <html>, before <body>. Never read theme from JS after the first paint.

09 / Stack Ports

Stack-Specific Porting Notes

Next.js on Vercel (primary)

N1
Font loading: Use next/font/local pointing at the WOFF2 files in public/assets/fonts/. Pass variable: '--font-inter' and display: 'swap'. Do not use next/font/google — fonts are self-hosted from our own origin, not fetched from Google Fonts.
N2
Images: Use next/image for all raster images. It handles AVIF/WebP conversion, srcset generation, and lazy loading automatically. Set priority={true} on the LCP image only — this is next/image's equivalent of fetchpriority="high".
N3
Theme script: In app/layout.tsx, add the theme-reading script as a <script dangerouslySetInnerHTML> with id="theme-script" inside <html>, before <body>. This prevents FOUC on page load.
N4
Tokens: Import styles/tokens.css in the root layout, not in individual components. Tokens are global and should only be loaded once.
N5
Metadata: Use the metadata export in each page file for Open Graph, canonical, and description. Never use raw meta tags in JSX for these — the metadata API handles deduplication correctly.

Squarespace Custom CSS

S1
Token injection: All CSS custom properties go in Design > Custom CSS. Put them inside :root {} at the top, before any selectors. Squarespace's own styles load before Custom CSS so tokens applied to :root are available globally.
S2
Font loading: Upload InterVariable.woff2 and JetBrainsMono[wght].woff2 as file assets, then add <link rel="preload"> and matching @font-face blocks in Settings > Advanced > Code Injection > Header. Point the src URLs at the uploaded file paths. No Google Fonts links. Use font-display: swap.
S3
Dark mode: Squarespace doesn't support data-theme natively. Add the theme toggle script in Code Injection > Footer. The data-theme attribute on document.documentElement works the same way.
S4
Components: Most of the system's components can be added via Code Blocks (injected HTML). For repeating components like nav, use Code Injection > Header for critical pieces. Performance will be lower than Next.js — Squarespace injects its own JS and can't be fully optimized.

Static HTML

The reference artifacts (01-04) are already static HTML and run without a build step. Copy the token block from 01-design-system.html to any new page's <style> block. For multi-page sites, extract tokens to a tokens.css file and link it as the first stylesheet. Everything else remains the same.

10 / Non-Negotiables

Non-Negotiable Rules

These are the things that must not drift during porting, refactoring, or feature additions. If a PR violates any of these, it does not merge.

01
Never hardcode a color. Every color value in a component must reference a CSS custom property from the token map. If you need a color that doesn't have a token, add the token first.
02
One h1 per page, no heading level skips. H1 is the page title. Never jump from H1 to H3. Never use headings for visual sizing — use CSS classes for that. Heading hierarchy is document structure, not visual hierarchy.
03
All interactive elements have visible focus rings. The 2.5px crimson ring with 3px offset is the standard. MaaS uses cyan instead. Never remove :focus-visible styling. Never use outline: none without a custom replacement that passes 4.5:1 contrast.
04
All form inputs have explicit label associations. for/id pairs, always. aria-label only as a last resort. Error messages use aria-invalid and aria-describedby. Never placeholder-only labeling.
05
Touch targets are minimum 44x44px. All buttons, links, checkboxes, and interactive elements. If visual size is smaller, use padding to meet the target without changing the visual size.
06
prefers-reduced-motion is always respected. The global reset in @layer base covers all animations. Any JS animation must also check window.matchMedia('(prefers-reduced-motion: reduce)').matches before running.
07
Core content works without JavaScript. Navigation links, all body copy, images, and primary CTAs must be functional without JS. Enhancements like mobile menu open/close, theme toggle, and FAQ accordion are progressive — the page must not be broken without them.
08
Every page has: canonical, meta description, og:title, og:description, og:image (1200x630), og:url, twitter:card. These are not optional for SEO pages. Generate them per-page, not globally with the same content on every page.
09
The logo is always <img src="/assets/brand/logo-d.png">. The canonical asset is a 1,425x1,425 PNG — raster is the source of truth, no SVG variant exists, and the mark must never be regenerated from a prompt. Set explicit width/height, use meaningful alt ("Dade Studio") when the logo stands alone, and put alt="" on the image with aria-label on the parent link only when the link text already names the site. Never a background-image.
10
Lighthouse 100/100/100/100 before merge. Not 98. Not "mostly 100". All four scores on mobile simulation. Run it on the deployed preview URL, not localhost. Lighthouse scores differ between local and CDN-served builds.
11
Dark mode must be genuinely designed, not inverted. Dark mode surfaces use the [data-theme="dark"] token overrides. Never use filter: invert() as a dark mode implementation. Images that look wrong in dark mode need a dark-specific <source media="(prefers-color-scheme: dark)"> variant.
12
No inline styles for recurring patterns. Inline styles are acceptable for one-off positioning tweaks or dynamic values (progress bar width, etc.). They are not acceptable for colors, fonts, spacing, or anything that should be a token or component class. If you write the same inline style twice, it becomes a class.
You're ready to build
This document plus the four reference artifacts contain everything needed to implement the system correctly. When in doubt: check the design system reference page first, look for the component, read the a11y note, then build. If something isn't in the system, add it to the system before shipping it.
